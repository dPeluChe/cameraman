## Import video into timeline (tester request)

- Import button accepts movies; each import = its own `.video` track row (distinct color per row, 6-color palette).
- Engine: overlay video tracks composited like the camera track (aspect-fit, post-zoom); **embedded audio extracted** into the mix (was silent) — preview and export share the pipeline.
- Chips: drag to move with **magnetic snap** (edges of any row's clips, segments, playhead, 0), edge trim clamped to real asset duration, click → PiP inspector (3×3 grid, S/M/L, fullscreen, remove), context menu (Split at Playhead, Jump to End, Place After Track Above, Place at Start, Move Row Up/Down, Remove).
- **Split** works on imported clips (toolbar ⌘B and chip menu).
- Rows reorder via context menu (also swaps compositing z-order); per-row eye/mute persists on the engine track (was muting all video rows at once).
- Lower-resolution import notice (upscale looks soft fullscreen; fine as PiP).

## Fixes

- **Rename/tags Save always failed**: alert dismissal reset the text field before the async save read it; value now captured first. Open editors reload after external rename (autosave used to write the old name back).
- **Mixed-resolution merged projects**: screen frames (and camera PiP) refit per-frame when the source resolution changes mid-track; mixed-res projects always route through the compositor (preview + export). Was: section B rendered tiny in a corner.
- **Ghost empty tracks** (left by removeClip) corrupted "Place After Track Above" — empty non-primary tracks now removed with their last clip.
- **Preview toggles audit**: of 7 checkboxes only Zoom worked. Overlays/Layout/Captions removed (were wired to nothing — backlog item to implement for real); Cursor/Clicks/Keys rebound to the real flags (telemetry overlay now functional). All grouped in a **View** menu on the timeline toolbar.
- **Auto-zoom hidden** behind `FeatureFlags.autoZoom` (default off) and no longer auto-applies on open — confused testers.
- **Reveal export in Finder** failed under sandbox (`NSWorkspace.open(folder)`) → `activateFileViewerSelecting`.
- **Check for Updates** exposed in the app Help menu (existed but only in the status-bar menu).
- PiP inspector grid buttons overlapped (bordered-style padding in 16pt cells) → flat cells.
- Default project names: **"MMM d · Word"** (e.g. "Jun 10 · Falcon") instead of "Untitled Recording".

## Export quality & project bundles (0.6.4 round, post-validation)

- **Preset bitrates were decorative**: the session was always created with `AVAssetExportPresetHighestQuality` (8-minute export → 951MB) and HEVC presets actually produced H.264. Fixed: session preset follows the codec, `fileLengthLimit` targets the preset's (video+audio) bitrate × duration, and a **quality picker** (Smaller 0.6× / Standard / Higher 1.5×) scales it. The size estimate uses the same formula, so it's finally honest.
- **Project bundles**: context menu *Export Bundle…* writes a portable `<Name>.cameramanproject` folder (essentials only — no cache/proxies/renders); "+" menu *Import Project…* brings one back under a fresh project id. `Project.withNewIdentity` extracted (shared with duplicateProject). 3 unit tests.
- **Split-vs-drag chip UX**: invisible 8pt trim handles intercepted body drags on short clips ("moving felt like extending"). Handles now exist only on the SELECTED chip and are visibly drawn — drag anywhere moves, click-then-grip trims.
- Codec coverage decision: H.264 + HEVC + GIF covers the beta; ProRes trivially addable on demand, VP9/AV1 out (no native AVFoundation support).
