## Export quality & project bundles (0.6.4 round, post-validation)

- **Preset bitrates were decorative**: the session was always created with `AVAssetExportPresetHighestQuality` (8-minute export → 951MB) and HEVC presets actually produced H.264. Fixed: session preset follows the codec, `fileLengthLimit` targets the preset's (video+audio) bitrate × duration, and a **quality picker** (Smaller 0.6× / Standard / Higher 1.5×) scales it. The size estimate uses the same formula, so it's finally honest.
- **Project bundles**: context menu *Export Bundle…* writes a portable `<Name>.cameramanproject` folder (essentials only — no cache/proxies/renders); "+" menu *Import Project…* brings one back under a fresh project id. `Project.withNewIdentity` extracted (shared with duplicateProject). 3 unit tests.
- **Split-vs-drag chip UX**: invisible 8pt trim handles intercepted body drags on short clips ("moving felt like extending"). Handles now exist only on the SELECTED chip and are visibly drawn — drag anywhere moves, click-then-grip trims.
- Codec coverage decision: H.264 + HEVC + GIF covers the beta; ProRes trivially addable on demand, VP9/AV1 out (no native AVFoundation support).

## MCP automation server + transcription/effects (0.7.0, branch `feat/transcription-and-mcp-settings`)

> Goal: drive Cameraman end-to-end from an AI assistant, and ship the supporting UI. Validated live via the MCP binary against the app's sandboxed projects (real `demo6_mcp.mp4` render: recording → split + sepia → color/image cards → text+arrow overlays → circular-PiP canvas).

### Settings
- **Transcription** tab: Whisper model picker (base/small/medium/large with sizes), gated by `TranscriptionEngine.isAvailable` (Apple Silicon).
- **Integrations** tab: the `cameraman-mcp` binary is **built, signed (hardened runtime) and embedded** in the `.app` at `Contents/Helpers/` via an Xcode build phase (`scripts/embed-mcp.sh`, `ENABLE_USER_SCRIPT_SANDBOXING = NO`), and auto-detected. Per-client registration snippets in **segmented tabs** (Claude Desktop / Claude Code / Codex) with Copy buttons. Server is pointed at the app's real projects via `CAMERAMAN_PROJECTS_DIR` (the app is sandboxed; the CLI defaulted to a different empty folder).

### MCP server — 42 tools
- Built on EngineKit so MCP edits behave exactly like the GUI (`EditorModel.mutate`, `OverlayEngine`, `ProjectLibrary`). Coverage: projects/management (list/get/create/duplicate/rename/tags/search/merge/bundle/delete), recording, clips (`add_clip`, split, delete, `edit_clip` = move/cross-track/retime/trim, `delete_range`, clip audio mute), adjustments (add/update/remove/clear/list), tracks (add/remove/move/`set_track`), overlays (`add_overlay` + list/update/delete, draw-on/fade-in), canvas (layout/background), delivery (`export_project` + jobs, `transcribe_project` + `get_captions`, local `suggest_*`).
- **Validation hardening** (post first runtime test): `add_clip`/asset tools copy the file into the project and reject nonexistent paths (no phantom clips); `set_track` reports the stored/clamped value; adjustment kinds validated against `AdjustmentKind.allBuiltIn`; `delete_project` added.
- **Consolidation pass** (50 → 42): merged per-type add/edit/track tools into `add_clip`/`edit_clip`/`set_track`; shared `resolveClip`/`startedJob` helpers; catalog regrouped by area; every source file kept ≤ ~460 LOC.

### Per-clip effects UI
- *Effects* popover in the timeline inspector for a selected imported-video clip (add/remove color filters, blur, audio pitch; commit-on-release slider). Popover keeps the inspector one line tall so adding effects doesn't push the timeline. Rendered by the overlay compositor on imported `.video` clips.

### Live refresh
- `ProjectsDirectoryWatcher` polls the Projects dir mtime (~2s) so MCP-created/deleted projects appear without re-activating the window. (`DispatchSource.makeFileSystemObjectEventSource` is unavailable on the Swift 6.2 toolchain here — verified — hence the mtime poll.) `ProjectStore.baseDirectory` made `public nonisolated` as the single source of truth.

### Export fixes (found during MCP validation)
- **Import-only projects** (no recording; image/color cards on overlay tracks) now compose and export: static clips are collected from overlay video tracks, the primary track is padded to span the timeline, and a static-clip instruction path was added to the export (mirroring the preview). Verified: video-only and video+card projects export.
- **Readable errors**: `ExportError` conforms to `LocalizedError`; the MCP surfaces thrown errors via `localizedDescription` (was "ExportError error 5" / bare case name).
- **Pure-static rejected up front**: `Project.hasRenderableVideo` guards `export()`/`exportGIF()` with an actionable `noVideoFrames` message instead of failing mid-render. Full base-track synthesis deferred (see `TASK_TODO.md`).
- **`delete_range` ripples across all tracks** (was primary-only) and fixes a split-position bug when the range sits inside a clip.

### Shared-helper cleanup (`/simplify`)
- `TimelineClip.visualAdjustmentConfigs()` (used by `Project.adjustmentConfigs` and the overlay sources) and `ProjectLibrary.stageAsset(...)` (used by drag-drop import and the MCP) extracted to remove duplication.
