## chore(compat): macOS 13 fallback para onChange y onKeyPress (aa1d89e)

- [x] **`ViewCompatibility.swift`** nuevo: `View.onChangeCompat` envuelve la API de macOS 14+ (`onChange(of:_,_:)`) y cae al `onChange(of:perform:)` legacy en macOS 13.
- [x] **11 vistas migradas** a `onChangeCompat`: `BackgroundControlsView`, `ProjectEditorCenterPanel`, `RecordingControlView`, `RecordingSourceSelectorView`, `TelemetryOverlayView`, `TeleprompterWindow`, `TimelineView`, `ToastView`, `ZoomControlsView`, `ZoomControlsView+Subviews`.
- [x] **`spacePlaybackShortcut()`** envuelve `onKeyPress(.space)` con `#available(macOS 14.0, *)`. En macOS 13 el atajo simplemente no aplica (sin warning ni crash).
- [x] **`TeleprompterWindow`** anotado con `@MainActor` para satisfacer al checker estricto.

Resultado: warnings de deprecación de macOS 14 silenciados sin subir el deployment target.

## Distribución beta

- DMG `Cameraman-beta-0.5.1.7.dmg` (5.7MB, universal arm64+x86_64) enviado a testers el 2026-05-10.
- Feedback: en **macOS Tahoe (26.4.1)** la app no abría aunque tuvieran "Allow apps from anywhere" activado. Causa: Gatekeeper endurecido en Tahoe sigue bloqueando bundles con `com.apple.quarantine` aunque la política global lo permita. Workaround documentado: `xattr -dr com.apple.quarantine /Applications/CameramanApp.app`. Mitigación futura propuesta: firma ad-hoc en `build-dmg.sh` (no pendiente aún).
