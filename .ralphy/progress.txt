**Zoom Transform Math:**
```
Focus Point (normalized): (focusX, focusY) where 0.0 ≤ focusX, focusY ≤ 1.0
Focus Point (render): (focusX * renderWidth, focusY * renderHeight)
Zoom Level: 1.0 = no zoom, 2.0 = 2x zoom, 5.0 = 5x zoom

Transform = BaseTransform × TranslateToFocus × Scale × TranslateFromFocus
Where:
  - TranslateToFocus: (focusX * renderWidth, focusY * renderHeight)
  - Scale: (zoomLevel, zoomLevel)
  - TranslateFromFocus: (-focusX * renderWidth, -focusY * renderHeight)
```

**Features:**
- **Important Click Detection**: Filters clicks based on:
  - Movement distance from previous click (default 50px minimum)
  - Button type (left/right/other clicks configurable)
  - Time since previous click (clustering detection)
- **Temporal Window Grouping**: Groups clicks into windows based on:
  - Time window size (default 2 seconds)
  - Minimum clicks per window (default 2 clicks)
  - Maximum click interval within window (default 1 second)
- **Window Analysis**: Calculates for each window:
  - Center point (average of all click positions)
  - Bounding box (min/max X/Y coordinates)
  - Click count and duration
  - Importance score (based on click density and cluster tightness)
- **Statistics Tracking**: Provides comprehensive statistics:
  - Total events processed
  - Total clicks and important clicks detected
  - Window count and clicks per second
  - Time range covered by telemetry
- **Multi-Monitor Support**: Tracks display ID for each click (optional)
- **File I/O**: Reads JSONL telemetry files with proper error handling
- **Async/Thread-Safe**: Actor-based implementation for thread safety

**Important Click Detection Logic:**
1. Filters click events (down events only)
2. Filters by button type (left/right/other based on configuration)
3. Calculates time since previous click
4. Calculates distance from previous click (Euclidean distance)
5. Creates new window ID when click interval exceeds maxClickInterval
6. Filters clicks that don't meet minimum movement distance (except first click in window)
7. Tracks window ID, display ID, and metadata for each important click

**Temporal Window Grouping Logic:**
1. Groups important clicks by time proximity
2. Creates windows when clicks are within timeWindowSize (default 2s)
3. Finalizes windows when they exceed time window or have minClicksPerWindow
4. Calculates window metadata:
   - Center point: Average X and Y of all clicks
   - Bounding box: Min/max X and Y of all clicks
   - Importance score: Click density + cluster tightness

**Importance Score Calculation:**
- Click density: Clicks per second in window
- Cluster tightness: Clicks per pixel area (normalized)
- Higher score = more important (more clicks, shorter duration, tighter cluster)
- Formula: `score = (clickDensity * 10) + clusterTightness`

**Configuration Presets:**
- **Default**: Balanced detection (2s window, 2 min clicks, 1s max interval, 50px min movement, left clicks only)

- **Conservative**: Less detection (1.5s window, 3 min clicks, 0.75s max interval, 80px min movement, left clicks only)
